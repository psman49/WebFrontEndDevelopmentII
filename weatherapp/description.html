Description of Personal Project- Weather App

        I chose to build a weather app for my personal project because I am blessed to live in the state of Utah 
        where we get to witness all the weather seasons throughout the year.The app has a search area where you 
        can type in any city in the world and see the weather for that city. I also dessigned it so that it is simple
        for anyone to use.

How to use the Weather App

        In that search area, where you see the word "search", type in any name of the city you want to check the weather
        for in the world. Let us say for example Sandy, you will then hit the search icon to your right, it looks like a 
        tiny frying pan. Once you hit that button, it will bring up the weather results of sandy at that current time.
        It will display the name of the city, how cold it is in Celcius and give a weather description, if it is snowing, 
        sunny, rainny or overcast. You will be able to see the humidity and wind speed of that city as well.

Javascript used in my project

        API- I used the open weather map api(https://openweathermap.org/api) to be fetch the current data weather for any given city.
        I used the fetch method (fectch()) to be to fetch the resouces(city weather) asynchronously across the network.
        With fetch() api we get a a json response and if you are to console.log it you see that there is all this data we can extract 
        out of this response. In my code you can see the I use destructuring assignment syntax to unpack the name,icon,description,humidity 
        and wind speed for the given city.

Third-Party APIs

        I used the openweather map API and with the usage of the fetch() I passed in the url -> https://api.openweathermap.org/data/2.5/weather?q=" +
        city + "&units=metric&appid=" + this.apiKey. Where you clearly see that i use an apikey, I got when i signed up and from the url you can see
        with the logic of destructing, I was able to unpack the city from the json that we get back when we hit the /weather endpoint and pass 
        in the respective parameters like city, name and api key to retrieve the current weather data for a given city.

JSON

        The response that is got back as asynchronous api call is made with the fetch(), we get back a reponse and if ok,we get the json object data 
        which I unpack using the destructing syntax so that I can extract the data for the city, description, name,speed, temperature and humidity.
        All of this is found in the script.js file.

Events

        You can see that I use the addEventListener, which listens to a click and executes the function to do a weather search.
        And clearly if a response is matched with the city name after the api call is made, we see current weather data for that city
        displayed.
